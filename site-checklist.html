bot引き継ぎ前チェック

botに任せる前に確認すること

Telegram botへ引き継ぐ前に、どこまで自動化できるか、どこは人間確認が必要かを整理します。

項目 botに任せられる可能性 人間確認が必要な理由 優先度
新規HTMLファイル作成 可能 GitHub API連携とテンプレート化が必要。 高
既存HTMLファイル更新 可能 上書きミス防止、差分確認、バックアップが必要。 高
リンク切れチェック 可能 GitHub Pages反映後のURL確認ロジックが必要。 高
カジノ情報の外部検索 一部可能 公式情報の真偽確認、規約変更、対象地域確認は慎重に扱う必要あり。 中
ボーナス情報の自動更新 慎重に実施 ボーナス内容は頻繁に変わるため、公式確認と日付管理が必要。 中
公開前の最終判断 人間確認推奨 表現、法的リスク、誇大表現、リンク先確認が必要。 高

運営上の注意

このサイトはオンラインカジノの比較メディアとして、条件確認・リスク理解・公式規約確認を重視します。 今後botへ作業を引き継ぐ場合も、以下の方針は維持してください。

  • 勝利保証、損失回収、射幸心を煽る表現を避ける
  • ボーナス条件、出金条件、本人確認、対象地域の確認を必ず入れる
  • 公式情報や規約が変わる前提で、断定しすぎない
  • 新規レビューを追加するときは、個別ページをまとめて作ってから一覧ページを一括更新する
  • 既存ロゴ画像を削除しない

チェックが終わったら、bot引き継ぎ仕様書へ。

まずはこのページでリンク切れ・画像・導線を確認し、問題なければ次に bot-handoff.html を作成して、Telegram botへ任せる作業範囲を整理します。
